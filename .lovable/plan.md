

# Fix: Sitemap Sub-URLs Returning 404

## Problem

The static `public/sitemap.xml` lists sub-sitemaps like `https://legallyspoken.com/sitemap-core.xml`, but these don't exist. The actual content is generated by the `generate-sitemap` edge function at URLs like:
```
https://<project>.supabase.co/functions/v1/generate-sitemap?type=core
```

There's no mapping between the public-facing `/sitemap-*.xml` URLs and the edge function.

## Fix

Update `public/sitemap.xml` to point sub-sitemap `<loc>` URLs directly at the edge function with the correct `?type=` parameter. Also update `public/robots.txt` to point to the edge function URL for the sitemap index.

The edge function already returns proper XML with `Content-Type: application/xml`, so Google will accept it.

| `https://legallyspoken.com/sitemap-core.xml` | `https://fpdfibyywvlcqjrkuuhz.supabase.co/functions/v1/generate-sitemap?type=core` |
| `https://legallyspoken.com/sitemap-tools.xml` | `...?type=tools` |
| `https://legallyspoken.com/sitemap-legal-terms.xml` | `...?type=legal-terms` |
| `https://legallyspoken.com/sitemap-guides.xml` | `...?type=guides` |

| `https://legallyspoken.com/sitemap-lawyers.xml` | `...?type=lawyers` |
| `https://legallyspoken.com/sitemap-blog.xml` | `...?type=blog` |

Also update `public/robots.txt` Sitemap directive to point to the edge function index (`?type=index` or no param).

And update the `sitemapIndex()` function inside the edge function itself to use the edge function URLs instead of `legallyspoken.com/sitemap-*.xml`.
